## 二、选取的核心业务模块：答题卡作业/测验系统

### 2.1 模块定位

教师端App的核心功能是帮助教师高效地布置和管理学习任务。目前支持的任务类型包括：

| 任务类型 | 说明 |
|---------|------|
| 作业和测验 | 教师布置给学生完成的练习或考试 |
| 资源 | 课件、视频等学习资料的下发 |
| 课程 | 结构化的课程内容推送 |

其中，**作业和测验**是使用频率最高的功能，而我主要负责的**答题卡类型**是作业和测验的一个重要子类型。

### 2.2 答题卡的两种评分模式

答题卡任务支持两种评分模式，适用于不同的教学场景：

**正确率模式**
- 适用场景：日常练习、课后作业
- 统计方式：只看对错，计算正确率
- 报告展示：正确率、错题数量、答对人数/答错人数

**计分模式**
- 适用场景：单元测试、期中期末考试
- 统计方式：每道题设置分值，计算得分
- 报告展示：总分、平均分、得分率、排名
- 特殊规则：多选题支持"少选得一半分"或"少选得N分"

## 三、业务流程与数据流转逻辑

### 3.1 整体业务流程

答题卡模块的完整业务流程可以分为五个阶段：

```
创建答题卡 → 布置任务 → 学生作答 → 教师批改 → 查看报告
```

### 3.2 各阶段详细流转

**阶段一：创建答题卡（我的资源模块）**

| 步骤 | 操作 | 数据变化 |
|------|------|---------|
| 1 | 教师进入"我的资源"，点击新建答题卡 | - |
| 2 | 填写基本信息：名称、年级、学科 | 生成答题卡基础信息 |
| 3 | 添加题目分组（如：一、选择题） | 创建分组结构 |
| 4 | 在分组下添加具体题目 | 记录题型、选项、正确答案 |
| 5 | 选择评分模式 | 正确率模式或计分模式 |
| 6 | 如果是计分模式，设置每题分值 | 记录各题分值 |
| 7 | 保存答题卡 | 答题卡资源入库，状态为"未下发" |

支持的题型：
- 选择题（单选/多选）：2-8个选项可配置
- 填空题：支持多个空，每空独立判断
- 解答题：需要教师人工批改

**阶段二：布置答题卡任务**

| 步骤 | 操作 | 数据变化 |
|------|------|---------|
| 1 | 教师进入"布置任务" | - |
| 2 | 选择任务类型：作业或测验 | 确定taskType |
| 3 | 选择布置对象：班级/学生 | 记录studentGroups |
| 4 | 选择要布置的答题卡 | 关联answerSheetId |
| 5 | 填写任务名称 | 记录taskName |
| 6 | 设置截止时间 | 记录deadline |
| 7 | 设置答案发布策略 | 立即/提交后/指定时间可见 |
| 8 | 确认布置 | 生成任务记录，答题卡状态变为"已下发" |

布置完成后：
- 任务推送至学生端
- 该答题卡被锁定，不可再编辑或删除

**阶段三：学生作答**

| 步骤 | 操作 | 数据变化 |
|------|------|---------|
| 1 | 学生在学生端收到任务 | - |
| 2 | 进入答题界面，逐题作答 | - |
| 3 | 作答方式：选择/文字输入/图片/手写 | - |
| 4 | 提交答案 | 记录学生答案、作答用时、提交时间 |

**阶段四：教师批改**

| 步骤 | 操作 | 数据变化 |
|------|------|---------|
| 1 | 教师进入"作业管理" | - |
| 2 | 点击进入待批改任务 | 加载学生作答数据 |
| 3 | 客观题（选择题）系统自动判分 | 自动生成对错结果 |
| 4 | 主观题（填空/解答）教师人工批改 | 记录教师判定：正确/错误/部分正确 |
| 5 | 如果是计分模式，主观题还需打分 | 记录各题得分 |
| 6 | 批改过程自动保存草稿 | 临时记录，防止丢失 |
| 7 | 全部批改完成，提交结果 | 批改记录正式入库 |

批改支持两种模式：
- 按题批改：选一道题，看所有学生的这道题答案
- 按人批改：选一个学生，看这个学生的所有题答案

**阶段五：查看报告**

| 统计维度 | 正确率模式展示 | 计分模式展示 |
|---------|---------------|-------------|
| 班级整体 | 平均正确率、完成进度 | 平均分、完成进度 |
| 学生个人 | 正确率、错题数 | 得分、排名 |
| 题目分析 | 正确人数/错误人数 | 平均分、得分率 |

报告支持导出为Excel文件。

### 3.4 关键业务规则

**答题卡状态管理：**
- 未下发状态：可编辑、可删除、可复用
- 已下发状态：禁止编辑、禁止删除、可复用

**计分模式特殊规则：**
- 多选题少选策略：
  - 不开启：少选算错误，得0分
  - 少选得一半：答对部分选项，得该题分值的50%
  - 少选得N分：答对部分选项，得固定N分
- 分值支持0.5分精度

**批改进度保存：**
- 批改过程中自动保存草稿，防止意外丢失
- 下次进入自动恢复到上次批改的位置

**答案发布策略：**
- 立即可见：提交后学生立刻能看到答案
- 提交后可见：学生提交自己的答案后才能看答案
- 指定时间可见：到达设定时间后统一公布答案

## 四、参与测试的需求版本

### 4.1 教师端 v1.3 - 我的资源_答题卡

**测试范围：**
- 答题卡的创建、编辑、删除、复用功能
- 各种题型（选择/填空/解答）的配置
- 已下发答题卡的操作限制
- 列表的搜索、筛选、分页

**发现的主要问题：**
- 已下发答题卡点击编辑按钮后能进入编辑页面（应该禁止）
- 复用答题卡时部分字段未正确复制

### 4.2 教师端 v1.3 - 布置答题卡作业/测验任务

**测试范围：**
- 布置流程的完整性
- 多班级、多学生的布置场景
- 时间设置和答案发布策略
- 布置后任务的下发和状态变更

**发现的主要问题：**
- 未选择布置对象时，提示信息不够明确
- 某些边界时间设置导致任务状态异常

### 4.3 教师端 v1.4 - 答题卡2期（计分模式）

**测试范围：**
- 计分模式的开关和分值设置
- 少选得分策略的各种组合
- 0.5分精度的计算准确性
- 报告页在两种模式下的差异化展示
- 模式切换时的数据处理

**发现的主要问题：**
- 多选题少选得分计算存在精度问题
- 模式切换时分值清空的确认提示缺失
- 报告导出的Excel文件名不包含任务名称
